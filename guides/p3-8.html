Build Guide: AG-UI Streaming Interface

Companion build guide — pairs with the Phase 2B incident-response system (Monitoring / Orchestrator / Remediation agents on AgentCore)

1. What You'll Build

  • A React (Vite) frontend that connects to your Phase 2B multi-agent incident-response backend over an AG-UI-compliant Server-Sent Events (SSE) stream
  • A live, auto-scrolling timeline/chat view that renders TEXT_MESSAGE_* events (agent reasoning/response text) and TOOL_CALL_* events (e.g. check_service_health, restart_service) as they happen, agent-by-agent
  • A chat input box that lets the user manually trigger a new incident by sending a natural-language message ("simulate a checkout-service outage") which becomes a new AG-UI run
  • An Approve / Reject button pair that stays disabled until a human-in-the-loop interrupt event arrives from the LangGraph backend, then unlocks so the user can approve the pending restart_service remediation action directly from the browser instead of the terminal
  • Deliverables checklist:
    • frontend/ — Vite + React + TypeScript app using @ag-ui/client (or CopilotKit) for SSE consumption
    • backend/agui_server.py — FastAPI/Bedrock AgentCore Runtime endpoint that translates Strands/LangGraph execution into AG-UI events
    • A deployed, public URL (Vercel or Cloudflare Pages) you can link from your portfolio

2. Architecture

┌─────────────────────────────┐        SSE (text/event-stream)        ┌──────────────────────────────────────┐
│   React Frontend (Vite)     │ ─────────────────────────────────────▶│  AG-UI Endpoint (FastAPI on           │
│                              │◀──────────────────────────────────────│  Bedrock AgentCore Runtime)           │
│  ChatInput → POST /run       │        AG-UI event stream             │                                        │
│  EventTimeline (renders      │                                       │  agui_server.py                       │
│    TEXT_MESSAGE_*,           │                                       │   ├─ receives RunAgentInput            │
│    TOOL_CALL_*)              │                                       │   ├─ invokes LangGraph app.stream()    │
│  ApproveButton (enabled on   │                                       │   │    Monitoring → Orchestrator →     │
│    HITL interrupt event)     │                                       │   │    Remediation (interrupt before   │
│                              │                                       │   │    restart_service)                │
└─────────────────────────────┘                                       │   └─ maps LangGraph events → AG-UI     │
                                                                        │        Event objects, yields as SSE   │
                                                                        └──────────────────────────────────────┘
  • The frontend never talks to Strands/LangGraph directly — it only ever speaks the AG-UI protocol (a small set of typed JSON events over SSE). This is the same decoupling principle as MCP for tools and A2A for agent-to-agent.
  • The backend is the "AG-UI server" — it owns the agent loop and emits events as the loop progresses. It is a thin translation layer around the Phase 2B system you already built.
  • The Approve button works by having the frontend hold local React state that flips true only when a CUSTOM or STATE_DELTA event announces an interrupt payload; clicking it POSTs a follow-up run that resumes the LangGraph checkpoint.

3. Prerequisites

  • Phase 2B incident-response backend already working locally (Monitoring, Orchestrator, Remediation agents; LangGraph graph with an interrupt() before restart_service)
  • Node.js ≥ 20, npm or pnpm
  • Python ≥ 3.11, pip install fastapi uvicorn sse-starlette langgraph strands-agents ag-ui-protocol
  • An AWS account with Bedrock AgentCore Runtime access (or run the FastAPI server locally behind ngrok while developing)
  • A Vercel or Cloudflare Pages account for deployment
  • npm create vite@latest familiarity; basic React hooks (useState, useEffect, useRef)

The AG-UI Event Schema

AG-UI defines 16 event types across 5 categories. Every event is a JSON object with a type discriminator; the backend streams a sequence of these over SSE and the frontend reduces them into UI state.

CategoryEvent typesPurpose
LifecycleRUN_STARTED, RUN_FINISHED, RUN_ERROR, STEP_STARTED, STEP_FINISHEDBracket a run and its internal steps so the UI knows when to show/hide loading state
TextTEXT_MESSAGE_START, TEXT_MESSAGE_CONTENT, TEXT_MESSAGE_ENDStream an agent's natural-language message token-by-token, keyed by messageId
Tool CallsTOOL_CALL_START, TOOL_CALL_ARGS, TOOL_CALL_END, TOOL_CALL_RESULTStream a tool invocation's name/args as they're generated, then its result once executed
StateSTATE_SNAPSHOT, STATE_DELTASync shared application state (e.g. incident severity, current remediation step) between agent and UI using JSON Patch deltas
SpecialCUSTOM, RAWEscape hatch for app-specific events — this is where we carry the HITL interrupt signal
Key insight: AG-UI does not define a first-class "human-in-the-loop" event. You model it with CUSTOM (e.g. {"type":"CUSTOM","name":"hitl_interrupt","value":{...}}) or a STATE_DELTA that flips an awaiting_approval field to true. Pick one convention and use it consistently on both ends.

Step 1 — Wire the LangGraph backend to emit AG-UI events

Wrap the compiled LangGraph app so each internal event is translated into an AG-UI Event and yielded as SSE.

# backend/agui_server.py
import json, uuid
from fastapi import FastAPI, Request
from sse_starlette.sse import EventSourceResponse
from langgraph.checkpoint.memory import MemorySaver
from incident_graph import build_incident_graph  # your Phase 2B LangGraph app

app = FastAPI()
checkpointer = MemorySaver()
graph = build_incident_graph(checkpointer=checkpointer)

def agui_event(event_type: str, **fields) -> str:
    """Serialize one AG-UI event as an SSE data line."""
    payload = {"type": event_type, **fields}
    return json.dumps(payload)

async def run_incident_stream(thread_id: str, user_message: str, resume: dict | None = None):
    run_id = str(uuid.uuid4())
    yield agui_event("RUN_STARTED", runId=run_id, threadId=thread_id)

    config = {"configurable": {"thread_id": thread_id}}
    input_state = {"messages": [{"role": "user", "content": user_message}]} if not resume else None
    stream_input = resume if resume else input_state

    try:
        async for chunk in graph.astream(stream_input, config=config, stream_mode="updates"):
            for node_name, node_output in chunk.items():
                yield agui_event("STEP_STARTED", stepName=node_name)

                # Agent text output -> TEXT_MESSAGE_* triplet
                if text := node_output.get("agent_message"):
                    msg_id = str(uuid.uuid4())
                    yield agui_event("TEXT_MESSAGE_START", messageId=msg_id, role="assistant")
                    yield agui_event("TEXT_MESSAGE_CONTENT", messageId=msg_id, delta=text)
                    yield agui_event("TEXT_MESSAGE_END", messageId=msg_id)

                # Tool call -> TOOL_CALL_* events
                if tool_call := node_output.get("tool_call"):
                    tc_id = str(uuid.uuid4())
                    yield agui_event("TOOL_CALL_START", toolCallId=tc_id, toolCallName=tool_call["name"])
                    yield agui_event("TOOL_CALL_ARGS", toolCallId=tc_id, delta=json.dumps(tool_call["args"]))
                    yield agui_event("TOOL_CALL_END", toolCallId=tc_id)
                    yield agui_event("TOOL_CALL_RESULT", toolCallId=tc_id, content=json.dumps(tool_call["result"]))

                yield agui_event("STEP_FINISHED", stepName=node_name)

        # Check whether the graph paused on an interrupt() before restart_service
        snapshot = graph.get_state(config)
        if snapshot.next:  # non-empty => graph is paused waiting for human input
            yield agui_event(
                "CUSTOM",
                name="hitl_interrupt",
                value={"action": "restart_service", "threadId": thread_id,
                       "details": snapshot.tasks[0].interrupts[0].value},
            )
        else:
            yield agui_event("RUN_FINISHED", runId=run_id, threadId=thread_id)

    except Exception as exc:
        yield agui_event("RUN_ERROR", runId=run_id, message=str(exc))

@app.post("/run")
async def run(request: Request):
    body = await request.json()
    thread_id = body.get("threadId") or str(uuid.uuid4())
    return EventSourceResponse(run_incident_stream(thread_id, body["message"]))

@app.post("/resume")
async def resume(request: Request):
    """Called when the user clicks Approve — resumes the paused graph."""
    body = await request.json()
    from langgraph.types import Command
    decision = Command(resume={"approved": body["approved"]})
    return EventSourceResponse(run_incident_stream(body["threadId"], user_message=None, resume=decision))
Interrupt setup reminder: your Phase 2B Remediation Agent node must call langgraph.types.interrupt({"action": "restart_service", "service": service_name}) before actually invoking the restart_service tool. That's what produces snapshot.next being non-empty above.

export type AgUiEvent =
  | { type: "RUN_STARTED"; runId: string; threadId: string }
  | { type: "RUN_FINISHED"; runId: string; threadId: string }
  | { type: "RUN_ERROR"; runId: string; message: string }
  | { type: "STEP_STARTED"; stepName: string }
  | { type: "STEP_FINISHED"; stepName: string }
  | { type: "TEXT_MESSAGE_START"; messageId: string; role: string }
  | { type: "TEXT_MESSAGE_CONTENT"; messageId: string; delta: string }
  | { type: "TEXT_MESSAGE_END"; messageId: string }
  | { type: "TOOL_CALL_START"; toolCallId: string; toolCallName: string }
  | { type: "TOOL_CALL_ARGS"; toolCallId: string; delta: string }
  | { type: "TOOL_CALL_END"; toolCallId: string }
  | { type: "TOOL_CALL_RESULT"; toolCallId: string; content: string }
  | { type: "STATE_SNAPSHOT"; snapshot: Record }
  | { type: "STATE_DELTA"; delta: unknown[] }
  | { type: "CUSTOM"; name: string; value: unknown }
  | { type: "RAW"; payload: unknown };

Step 5 — Consume the SSE stream in a hook

// src/hooks/useAgUiStream.ts
import { useCallback, useRef, useState } from "react";
import type { AgUiEvent } from "../types/agui";

const AGUI_ENDPOINT = import.meta.env.VITE_AGUI_ENDPOINT ?? "http://localhost:8000";

export function useAgUiStream() {
  const [events, setEvents] = useState([]);
  const [pendingInterrupt, setPendingInterrupt] = useState<{ threadId: string; details: unknown } | null>(null);
  const threadIdRef = useRef(crypto.randomUUID());

  const consumeStream = useCallback(async (response: Response) => {
    const reader = response.body!.getReader();
    const decoder = new TextDecoder();
    let buffer = "";

    while (true) {
      const { value, done } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split("\n\n");
      buffer = lines.pop() ?? "";

      for (const line of lines) {
        const dataLine = line.split("\n").find((l) => l.startsWith("data:"));
        if (!dataLine) continue;
        const evt = JSON.parse(dataLine.slice(5).trim()) as AgUiEvent;
        setEvents((prev) => [...prev, evt]);

        if (evt.type === "CUSTOM" && evt.name === "hitl_interrupt") {
          const v = evt.value as { threadId: string; details: unknown };
          setPendingInterrupt({ threadId: v.threadId, details: v.details });
        }
      }
    }
  }, []);

  const triggerIncident = useCallback(async (message: string) => {
    setEvents([]);
    const res = await fetch(`${AGUI_ENDPOINT}/run`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ message, threadId: threadIdRef.current }),
    });
    await consumeStream(res);
  }, [consumeStream]);

  const approveRemediation = useCallback(async (approved: boolean) => {
    if (!pendingInterrupt) return;
    setPendingInterrupt(null);
    const res = await fetch(`${AGUI_ENDPOINT}/resume`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ threadId: pendingInterrupt.threadId, approved }),
    });
    await consumeStream(res);
  }, [pendingInterrupt, consumeStream]);

  return { events, pendingInterrupt, triggerIncident, approveRemediation };
}

Step 6 — Render the timeline, chat input, and Approve button

// src/components/EventTimeline.tsx
import type { AgUiEvent } from "../types/agui";

export function EventTimeline({ events }: { events: AgUiEvent[] }) {
  const messages = new Map<string, string>();
  const toolCalls = new Map<string, { name: string; args: string; result?: string }>();

  for (const e of events) {
    if (e.type === "TEXT_MESSAGE_CONTENT") {
      messages.set(e.messageId, (messages.get(e.messageId) ?? "") + e.delta);
    }
    if (e.type === "TOOL_CALL_START") toolCalls.set(e.toolCallId, { name: e.toolCallName, args: "" });
    if (e.type === "TOOL_CALL_ARGS") {
      const tc = toolCalls.get(e.toolCallId);
      if (tc) tc.args += e.delta;
    }
    if (e.type === "TOOL_CALL_RESULT") {
      const tc = toolCalls.get(e.toolCallId);
      if (tc) tc.result = e.content;
    }
  }

  return (
    <div className="timeline">
      {events.map((e, i) => {
        if (e.type === "STEP_STARTED") return <div key={i} className="step-marker">▶ {e.stepName}</div>;
        if (e.type === "TEXT_MESSAGE_END")
          return <div key={i} className="chat-bubble agent">{messages.get(e.messageId)}</div>;
        if (e.type === "TOOL_CALL_RESULT") {
          const tc = toolCalls.get(e.toolCallId);
          return (
            <div key={i} className="tool-card">
              <strong>🔧 {tc?.name}</strong>
              <pre>{tc?.args}</pre>
              <pre>→ {tc?.result}</pre>
            </div>
          );
        }
        if (e.type === "RUN_ERROR") return <div key={i} className="error-banner">{e.message}</div>;
        return null;
      })}
    </div>
  );
}
// src/components/ApproveButton.tsx
export function ApproveButton({
  pendingInterrupt,
  onDecision,
}: {
  pendingInterrupt: { threadId: string; details: unknown } | null;
  onDecision: (approved: boolean) => void;
}) {
  return (
    <div className="approve-bar">
      <button disabled={!pendingInterrupt} onClick={() => onDecision(true)} className="approve">
        ✅ Approve restart_service
      </button>
      <button disabled={!pendingInterrupt} onClick={() => onDecision(false)} className="reject">
        ❌ Reject
      </button>
      {pendingInterrupt && (
        <pre className="interrupt-details">{JSON.stringify(pendingInterrupt.details, null, 2)}</pre>
      )}
    </div>
  );
}
// src/App.tsx
import { useState } from "react";
import { useAgUiStream } from "./hooks/useAgUiStream";
import { EventTimeline } from "./components/EventTimeline";
import { ApproveButton } from "./components/ApproveButton";

export default function App() {
  const [draft, setDraft] = useState("");
  const { events, pendingInterrupt, triggerIncident, approveRemediation } = useAgUiStream();

  return (
    <div className="app">
      <h1>Incident Response — Live Feed</h1>
      <EventTimeline events={events} />
      <ApproveButton pendingInterrupt={pendingInterrupt} onDecision={approveRemediation} />
      <div className="chat-input">
        <input value={draft} onChange={(e) => setDraft(e.target.value)}
               placeholder="e.g. simulate a checkout-service outage" />
        <button onClick={() => { triggerIncident(draft); setDraft(""); }}>Trigger Incident</button>
      </div>
    </div>
  );
}

CORS: your FastAPI backend must send Access-Control-Allow-Origin for the deployed frontend's origin, and SSE responses must not be gzip-compressed by an intermediate proxy — both break streaming silently.

Verification

  • Run curl -N -X POST http://localhost:8000/run -H "Content-Type: application/json" -d '{"message":"simulate an outage"}' — you should see a stream of data: {...} lines print live, not all at once
  • Open the frontend, type an incident, click "Trigger Incident" — the timeline should populate step-by-step (not in one burst) as the Monitoring → Orchestrator → Remediation nodes execute
  • When the graph reaches the interrupt() before restart_service, the Approve/Reject buttons should flip from disabled to enabled without a page reload
  • Click Approve — a new SSE stream from /resume should complete the run and show RUN_FINISHED in the timeline (log it to the console for a quick check)

Troubleshooting

  • Stream arrives all at once instead of incrementally: a proxy (nginx, AWS ALB, or even uvicorn --workers with certain buffering configs) is buffering the response. Set X-Accel-Buffering: no and confirm EventSourceResponse isn't wrapped by gzip middleware.
  • Approve button never enables: check that your LangGraph node actually calls interrupt() (not just returns early) and that graph.get_state(config).next is non-empty after the stream ends — print it server-side to confirm before debugging the frontend.
  • CORS errors in the browser console: add from fastapi.middleware.cors import CORSMiddleware and allow your Vercel/Pages origin explicitly; wildcard * does not work with credentialed SSE requests in some browsers.
  • Events arrive out of order or duplicated: make sure you're using one persistent thread_id per incident and not creating a new LangGraph checkpoint thread on every keystroke or reconnect.
