Build Guide: Typed Research Agent (PydanticAI)

Project 1A companion guide · No AWS required — Anthropic API direct · PydanticAI v2.0

What You'll Build

  • A CLI tool, research_agent.py, that takes a topic string and returns a schema-validated ResearchBrief
  • Three agent tools: web_search(query), fetch_page(url), summarize_source(text)
  • An orchestration loop: search → fetch top 3 results → summarize each → synthesize into ResearchBrief
  • A PydanticAI output_validator that raises ModelRetry when the output violates business rules (e.g. wrong word count), forcing an automatic re-generation
  • Exact deliverables:
    • research_agent.py — the agent, tools, and CLI entry point
    • tests/test_agent.py — smoke test asserting the OUTPUT SCHEMA is valid
    • sample_output.json — a committed real run, for portfolio review
    • requirements.txt

Architecture

topic string
     │
     ▼
 ┌─────────────────────────────────────────┐
 │  PydanticAI Agent (Claude Sonnet 4.5)    │
 │  output_type = ResearchBrief             │
 └───────────────┬───────────────────────────┘
                 │ reasons, picks tools
     ┌───────────┼────────────────┐
     ▼           ▼                ▼
 web_search   fetch_page    summarize_source
 (DuckDuckGo) (httpx +      (nested LLM call,
              trafilatura)   Claude Haiku)
     │           │                │
     └─────┬─────┴────────┬───────┘
           ▼              ▼
     top 3 URLs      per-source digest
           │              │
           └──────┬───────┘
                  ▼
      agent synthesizes final answer
                  ▼
      output_validator checks word count,
      raises ModelRetry on failure
                  ▼
      validated ResearchBrief (JSON)
  • The orchestrator agent (Sonnet) does the planning and final synthesis; it never reads a full raw page itself — it delegates that to summarize_source
  • summarize_source is a second, cheaper PydanticAI Agent (Haiku) invoked as a nested LLM call inside a tool — this is the "sub-agent as tool" pattern you'll reuse constantly in later phases
  • Validation happens at two layers: Pydantic field constraints (structural — types, list lengths) and an output_validator (semantic — word count range), the second one able to trigger a retry loop

Step 2 — Define the ResearchBrief schema

This is the contract the agent must satisfy. Put it at the top of research_agent.py.

from typing import Literal
from pydantic import BaseModel, Field, HttpUrl


class ResearchBrief(BaseModel):
    summary: str = Field(
        ..., description="A 150-250 word synthesis of the research findings."
    )
    key_findings: list[str] = Field(
        ..., min_length=3, max_length=5,
        description="3-5 concise, standalone bullet-point findings."
    )
    sources: list[HttpUrl] = Field(
        ..., min_length=2,
        description="URLs of pages actually fetched and used."
    )
    confidence: Literal["low", "medium", "high"] = Field(
        ..., description="Overall confidence in the findings given source quality/quantity."
    )
Why this matters: min_length/max_length on the lists and HttpUrl on sources are structural checks Pydantic enforces before your code ever sees the object — malformed output never reaches your caller.

Step 3 — Build the three tools

web_search — thin wrapper over DuckDuckGo (swap for Tavily's REST API if you have a key; the tool signature stays identical):

import httpx
import trafilatura
from duckduckgo_search import DDGS
from pydantic_ai import Agent, RunContext, ModelRetry


def web_search(query: str, max_results: int = 5) -> list[dict]:
    """Search the web for a query. Returns a list of
    {title, href, body} result dicts, ranked by relevance."""
    with DDGS() as ddgs:
        return list(ddgs.text(query, max_results=max_results))

fetch_page — downloads a URL and strips nav/ads/boilerplate down to article text: 

def fetch_page(url: str) -> str:
    """Fetch a URL and extract its main readable text content,
    discarding navigation, ads, and boilerplate. Returns plain text
    (may be empty if extraction fails)."""
    resp = httpx.get(
        url, timeout=10.0, follow_redirects=True,
        headers={"User-Agent": "research-agent/0.1 (+https://staycurious.me)"},
    )
    resp.raise_for_status()
    text = trafilatura.extract(
        resp.text, include_comments=False, include_tables=False
    )
    return text or ""

summarize_source — a nested LLM call. It's a second, smaller PydanticAI agent used purely as a summarization sub-routine: 

summarizer = Agent(
    "anthropic:claude-haiku-4-5",
    system_prompt=(
        "Condense the given document into a factual 3-5 sentence "
        "summary. No opinions, no filler, no restating the prompt."
    ),
)


async def summarize_source(text: str) -> str:
    """Summarize a single fetched document's raw text into a short
    factual digest suitable for downstream synthesis."""
    truncated = text[:12000]  # keep well under the summarizer's context budget
    result = await summarizer.run(truncated)
    return result.output

Step 4 — Wire the tools into the orchestrator agent

import os
from pydantic_ai.models.anthropic import AnthropicModel
from pydantic_ai.providers.anthropic import AnthropicProvider

model = AnthropicModel(
    "claude-sonnet-4-5",
    provider=AnthropicProvider(api_key=os.environ["ANTHROPIC_API_KEY"]),
)

agent = Agent(
    model,
    output_type=ResearchBrief,
    system_prompt=(
        "You are a research assistant. Given a topic:\n"
        "1. Call web_search to find candidate sources.\n"
        "2. Call fetch_page on the top 3 most relevant results.\n"
        "3. Call summarize_source on each fetched page's text.\n"
        "4. Synthesize the summaries into a single ResearchBrief.\n"
        "Only cite sources you actually fetched via fetch_page. "
        "If fewer than 2 pages fetch successfully, retry with a "
        "different search query before giving up."
    ),
)

# Register the plain (non-context) tools
agent.tool_plain(web_search)
agent.tool_plain(fetch_page)
agent.tool_plain(summarize_source)
  • tool_plain registers a tool that doesn't need the PydanticAI RunContext — appropriate here since none of the three tools need agent-level dependencies
  • The system prompt encodes the orchestration plan explicitly; PydanticAI's model still decides tool call order and arguments at runtime

Step 5 — Add semantic validation with ModelRetry

Pydantic field constraints catch structural problems (wrong types, list too short). ModelRetry lets you catch semantic problems and hand the model specific feedback to fix them — no manual retry loop needed:

@agent.output_validator
async def validate_brief(
    ctx: RunContext[None], output: ResearchBrief
) -> ResearchBrief:
    word_count = len(output.summary.split())
    if not (150 <= word_count <= 250):
        raise ModelRetry(
            f"Your summary is {word_count} words. It must be between "
            f"150 and 250 words. Rewrite the summary to fit that range "
            f"while keeping all the same facts."
        )
    return output
How the retry loop works: when agent.run() parses the model's output into ResearchBrief and a validator (Pydantic field OR output_validator) raises an error, PydanticAI automatically appends the error message to the conversation and asks the model to produce a corrected output — up to a configurable retry limit (default 1, raise via Agent(..., output_retries=3)). You get self-correcting structured output for free.

Step 6 — CLI entry point

import argparse
import asyncio
import sys


def main() -> None:
    parser = argparse.ArgumentParser(
        description="Typed research agent — returns a validated ResearchBrief."
    )
    parser.add_argument("topic", help='e.g. "quantum error correction 2026"')
    parser.add_argument("-o", "--output", help="Write JSON result to this file")
    args = parser.parse_args()

    try:
        result = asyncio.run(agent.run(args.topic))
    except Exception as exc:
        print(f"Agent run failed: {exc}", file=sys.stderr)
        sys.exit(1)

    brief_json = result.output.model_dump_json(indent=2)
    print(brief_json)

    if args.output:
        with open(args.output, "w") as f:
            f.write(brief_json)


if __name__ == "__main__":
    main()

Step 7 — Write the smoke test

Assert the schema, not the content — LLM output varies run to run, but the contract must always hold:

# tests/test_agent.py
import json
import sys
import os

sys.path.insert(0, os.path.dirname(os.path.dirname(__file__)))

import pytest
from research_agent import ResearchBrief, agent


@pytest.mark.asyncio
async def test_research_brief_schema_valid():
    result = await agent.run("retrieval-augmented generation basics")
    brief = result.output

    assert isinstance(brief, ResearchBrief)
    word_count = len(brief.summary.split())
    assert 150 <= word_count <= 250
    assert 3 <= len(brief.key_findings) <= 5
    assert len(brief.sources) >= 2
    assert brief.confidence in ("low", "medium", "high")


def test_sample_output_file_matches_schema():
    """Guards against schema drift breaking a previously-committed sample."""
    with open(os.path.join(os.path.dirname(__file__), "..", "sample_output.json")) as f:
        data = json.load(f)
    ResearchBrief.model_validate(data)  # raises if invalid
pytest tests/ -v

Verification

  • Run python research_agent.py "topic of your choice" — expect a pretty-printed JSON object matching the ResearchBrief shape, printed to stdout in under ~30 seconds
  • Run pytest tests/ -v — both tests should pass; test_research_brief_schema_valid makes a real API call, so it needs ANTHROPIC_API_KEY set
  • Deliberately break the schema to confirm retries work: temporarily set the validator's range to 1000 <= word_count <= 1001 and rerun — you should see the model attempt a second generation before eventually failing with a UnexpectedModelBehavior once retries are exhausted
  • Confirm sources actually points at URLs your fetch_page tool was called with — cross-check against agent run logs (result.all_messages())

Troubleshooting

Empty text from fetch_page: some sites block generic user agents or render content client-side (JS-heavy SPAs). trafilatura can't execute JavaScript. Fix: add a fallback that skips empty extractions and tells the agent (via the tool's return value) to try the next search result instead.
DuckDuckGo rate limiting / empty results: duckduckgo-search hits IP-based throttling on rapid repeat queries during dev. Add a short time.sleep(1) between calls while testing, or switch to the Tavily API (paid tier has no such issue) by swapping only the body of web_search.
Agent exhausts retries and raises UnexpectedModelBehavior: means your output_validator feedback wasn't actionable enough, or output_retries is too low. Make the ModelRetry message more specific (state the exact numbers), and bump Agent(..., output_retries=3).
ANTHROPIC_API_KEY not found: PydanticAI's AnthropicProvider reads the key you pass explicitly — if you rely on the env var implicitly instead, confirm you exported it in the same shell session running the script, not a different terminal tab.
