PydanticAI Crash Course: Agentic Framework For Production

Neural Nine · Feb 2026 · YouTube ↗

What is PydanticAI?

  • Agentic framework built on top of the pydantic validation library — same team
  • Focus: type safety, structured output, production-readiness — "the FastAPI of agent frameworks"
  • LangChain = generalist/opinionated (like Django). PydanticAI = minimal/typed (like FastAPI)
  • Model-agnostic: OpenAI, Anthropic, Gemini, Ollama, etc. — swap the model string
  • Used by LangChain and LlamaIndex internally — pydantic validation is already everywhere

Basic Agent

from pydantic_ai import Agent

agent = Agent(
    "openai:gpt-4o",
    system_prompt="You are a helpful assistant. Always answer in one concise sentence."
)

# In Jupyter (async context)
response = await agent.run("What is Python?")
print(response.output)  # Python is a high-level interpreted programming language...

# In a .py script
response = agent.run_sync("What is Python?")
print(response.output)
async vs sync: In Jupyter use await agent.run(). In a script use agent.run_sync(). Forgetting await returns a coroutine object, not the result.

Structured Output

from pydantic import BaseModel
from pydantic_ai import Agent

class Person(BaseModel):
    name: str
    age: int
    job: str

agent = Agent(
    "openai:gpt-4o",
    system_prompt="You always respond with structured output for people.",
    output_type=Person  # enforces schema on the model's response
)

response = await agent.run("Mike is a 20-year-old plumber.")
print(response.output)  # Person(name='Mike', age=20, job='plumber')
print(type(response.output))  #
  • PydanticAI validates the LLM output against the schema — retries if validation fails
  • Key differentiator: type-safe structured output is a first-class feature, not a workaround

Streaming

# Stream text chunks as they're generated
async with agent.run_stream("What is Python?") as run:
    async for chunk in run.stream_text(delta=True):  # delta=True = incremental chunks
        print(chunk, end="", flush=True)
  • delta=False (default): each iteration gives the full accumulated text so far
  • delta=True: each iteration gives only the new chunk — use this for streaming to users

Conversation History

r1 = await agent.run("What is Python?")
# ... some time later ...

# Pass all_messages to include full conversation context
r2 = await agent.run(
    "When was it first released?",
    message_history=r1.all_messages()  # agent now knows we talked about Python
)

# new_messages() = only the last exchange (agent starts forgetting earlier context)
r3 = await agent.run("Tell me more.", message_history=r1.new_messages())
  • all_messages(): full history — agent remembers everything from the session
  • new_messages(): only the last response — agent forgets earlier turns

System Prompt vs Instructions

Key distinction: system_prompt travels with the conversation history when passed to other agents. instructions does NOT — it stays scoped to the agent that defined it.
# Agent 1 uses instructions (NOT transferred via message_history)
agent1 = Agent("openai:gpt-4o", instructions="If asked your fav color, say blue.")
r1 = await agent1.run("What is your favorite color?")

# Agent 2: passes agent1's history BUT gets its own instructions
agent2 = Agent("openai:gpt-4o", instructions="If asked fav color, say red.")
r2 = await agent2.run(
    "What is your favorite color?",
    message_history=r1.all_messages()
)
print(r2.output)  # "red" — agent2's instructions take precedence

# Switch agent1 to system_prompt → now it DOES bleed through history
agent1_sp = Agent("openai:gpt-4o", system_prompt="If asked fav color, say blue.")
  • Use instructions when you want clean agent isolation (sub-agent pattern)
  • Use system_prompt when you want the agent's personality to carry through handoffs

Tool Registration (Two Ways)

from pydantic_ai import Agent

agent = Agent("openai:gpt-4o", system_prompt="You are a helpful assistant.")

# Method 1: pass as list in Agent() constructor
def get_favorite_color(name: str) -> str:
    """Returns the favorite color of a user based on their name."""
    if name == "Florian": return "teal"
    elif name == "Mike": return "orange"
    return "no information found"

agent_v1 = Agent("openai:gpt-4o", tools=[get_favorite_color])

# Method 2: @agent.tool_plain decorator (no context needed)
@agent.tool_plain
def get_favorite_color_v2(name: str) -> str:
    """Returns the favorite color of a user based on their name."""
    if name == "Florian": return "teal"
    return "unknown"

response = await agent.run("What is Florian's favorite color?")
  • tool_plain: no agent context needed in the function — simplest option
  • tool: receives RunContext as first arg — use when you need to access dependencies
  • Type hints on parameters are required — PydanticAI uses them to generate the tool schema for the LLM

Dependency Injection

from pydantic_ai import Agent, RunContext
from dataclasses import dataclass
import sqlite3

@dataclass
class MyDeps:
    username: str
    db_connection: sqlite3.Connection

agent = Agent(
    "openai:gpt-4o",
    deps_type=MyDeps,
    system_prompt="You are a helpful assistant. Always call the user by name."
)

# Dynamic system prompt enhancement using the dependency
@agent.system_prompt
def add_username(ctx: RunContext[MyDeps]) -> str:
    return f"The user's name is {ctx.deps.username}."

# Tool that uses the database connection from context
@agent.tool
async def list_employees(ctx: RunContext[MyDeps]) -> list[tuple]:
    """List all employees from the database."""
    cursor = ctx.deps.db_connection.cursor()
    cursor.execute("SELECT name, age, job FROM person")
    return cursor.fetchall()

# Usage
conn = sqlite3.connect("data/database.db", check_same_thread=False)
deps = MyDeps(username="Florian", db_connection=conn)

response = await agent.run("Who are the employees?", deps=deps)
print(response.output)  # "Florian, here are the employees: Mike (20, plumber)..."
  • deps_type: type of the dependencies object — can be a simple type (int, str) or a dataclass
  • @agent.system_prompt: decorator to enhance the system prompt dynamically using context
  • ctx.deps: access the injected dependencies inside tools
  • Use check_same_thread=False for SQLite connections in async context
Pattern: DI lets you inject database connections, API clients, user context — anything that the agent or its tools need — without making them global state.

Tool Sets

from pydantic_ai.toolsets import FunctionToolset
from datetime import datetime

def get_current_date() -> str:
    """Returns today's date."""
    return datetime.now().strftime("%Y-%m-%d")

def get_current_time() -> str:
    """Returns the current time."""
    return datetime.now().strftime("%H:%M")

def get_current_weekday() -> str:
    """Returns the current day of the week."""
    return datetime.now().strftime("%A")

datetime_tools = FunctionToolset([get_current_date, get_current_time, get_current_weekday])

# Add toolset to agent
agent = Agent("openai:gpt-4o", toolsets=[datetime_tools])

# Or add per-run only
response = await agent.run("What time is it?", toolsets=[datetime_tools])
  • Toolsets group related tools for organizational clarity
  • You can add toolsets per-run — useful for conditionally giving an agent access to certain tools

Timeouts & Retries

import asyncio
from pydantic_ai import Agent

agent = Agent(
    "openai:gpt-4o",
    tool_timeout=5.0,   # seconds before a tool call is cancelled
    retries=3           # retry tool calls that timeout or fail
)

@agent.tool_plain
async def slow_tool() -> str:
    """A tool that takes 10 seconds."""
    await asyncio.sleep(10)
    return "blue"

# With timeout=5, the tool will be cancelled and retried up to 3 times
# After retries exhausted, agent reports it can't get the answer
  • Always set tool_timeout for tools that call external APIs — prevent infinite hangs
  • retries: total retry budget across all tool calls in one agent run

Built-in Tools

from pydantic_ai import Agent
from pydantic_ai.models.openai import OpenAIResponsesModel

# Web search (requires OpenAI Responses API)
from pydantic_ai import WebSearchTool

model = OpenAIResponsesModel("gpt-4o")
agent = Agent(model=model, builtin_tools=[WebSearchTool()])

response = await agent.run("What is the latest video from Neural Nine?")

# Code execution
from pydantic_ai import CodeExecutionTool
agent_code = Agent(model=model, builtin_tools=[CodeExecutionTool()])
response = await agent_code.run("What is the factorial of 24?")
print(response.builtin_tool_calls)  # shows the code that was executed
  • WebSearchTool: uses OpenAI's built-in web search — requires OpenAIResponsesModel
  • CodeExecutionTool: lets the agent write and run Python to answer questions

MCP Server Integration

from pydantic_ai import Agent
from pydantic_ai.mcp import MCPServerStreamableHTTP

server = MCPServerStreamableHTTP("http://localhost:8000/mcp")

agent = Agent(
    "openai:gpt-4o",
    toolsets=[server]  # agent discovers tools from MCP server automatically
)

result = await agent.run("What is my favorite color?")
print(result.output)  # Uses tool from MCP server
  • The agent auto-discovers all tools exposed by the MCP server at run time
  • Supports both stdio and HTTP transports
  • The same MCP server can serve Claude Desktop, Cursor, Strands, and PydanticAI simultaneously

Key Takeaways

  • PydanticAI's big wins over LangChain: cleaner code, first-class type safety, dependency injection
  • Structured output via output_type=MyModel is the framework's core strength — use it
  • Dependency injection is how you pass database connections, API clients, and user context cleanly
  • system_prompt vs instructions: use instructions for isolated sub-agents to prevent context bleed
  • Tool docstrings = what the LLM reads to decide when to call the tool — write them clearly
  • Always set tool_timeout and retries before going to production
