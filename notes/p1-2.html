Strands SDK: From First Agent to Production-Ready

AWS Community · Nov 2025 · YouTube ↗

Overview

Six progressive labs taking Strands from a basic agent to a production-deployed multi-agent system with observability. Each lab builds on the previous one.

  • Lab 1 → Basic agent setup
  • Lab 2 → Model provider switching
  • Lab 3 → AWS service access via use_aws tool
  • Lab 4 → MCP integration with FastMCP
  • Lab 5 → A2A multi-agent handoffs
  • Lab 6 → Observability with Langfuse + Ragas evaluation

Lab 1 — Basic Agent Setup

  • Install strands-agents and strands-agents-tools
  • Set up AWS credentials for Bedrock (default provider): aws configure or environment variables AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_DEFAULT_REGION
  • Create a minimal agent with a system prompt and call it
from strands import Agent

agent = Agent(system_prompt="You are a helpful assistant.")
agent("Hello!")
  • Default model: claude-sonnet on Bedrock (us-east-1 by default)
  • Streaming output is on by default — the response prints to terminal as it's generated

Lab 2 — Model Providers

  • Strands is model-agnostic — swap the model parameter to change providers
  • Bedrock supports multiple model families; you can also use Anthropic API, OpenAI, Ollama
from strands import Agent
from strands.models import BedrockModel, AnthropicModel

# Bedrock with specific model ID
bedrock_agent = Agent(model=BedrockModel(
    model_id="us.anthropic.claude-sonnet-4-5-20251001-v1:0"
))

# Anthropic API directly (requires ANTHROPIC_API_KEY)
anthropic_agent = Agent(model=AnthropicModel(
    model_id="claude-opus-4-8",
    api_key="your-key"
))
  • Use Bedrock for AWS-native deployments (IAM, VPC, no key management)
  • Use Anthropic/OpenAI direct for local dev when you don't have an AWS account configured

Lab 3 — AWS Services via use_aws

  • The use_aws tool lets your agent call any AWS SDK operation using natural language
  • The LLM translates your intent into the correct boto3 API call
from strands import Agent
from strands_tools import use_aws

agent = Agent(
    system_prompt="You are an AWS assistant. Use AWS tools to help the user.",
    tools=[use_aws]
)

agent("List all S3 buckets in my account")
agent("Get the last 5 CloudWatch log events from /aws/lambda/my-function")
  • Requires the agent's IAM role/user to have appropriate permissions
  • Works for Bedrock KB queries, DynamoDB reads/writes, Lambda invocations, and more
  • Key pattern for building internal tooling that operates on your own AWS resources

Lab 4 — MCP Integration with FastMCP

  • MCP (Model Context Protocol) lets your agent discover and use tools hosted on external servers
  • FastMCP is a Python library for building MCP servers quickly
# Server side (run separately): server.py
from fastmcp import FastMCP

mcp = FastMCP("calculator")

@mcp.tool()
def add(a: float, b: float) -> float:
    """Add two numbers together."""
    return a + b

mcp.run(transport="stdio")

# Agent side: client.py
from strands import Agent
from strands.tools.mcp import MCPClient

client = MCPClient(["python", "server.py"])

with client:
    agent = Agent(
        system_prompt="You are a math assistant.",
        tools=client.list_tools()  # discovers tools from MCP server
    )
    agent("What is 42 + 58?")
  • MCP separates tool implementation (server) from agent (client) — enables reuse across frameworks
  • The agent automatically discovers what tools are available and their schemas
  • Also supports HTTP transport for remote MCP servers
Portfolio note: Any MCP server you build is reusable by Claude Desktop, Cursor, and any MCP-compatible agent — not just Strands.

Lab 5 — A2A Multi-Agent Handoffs

  • A2A (Agent-to-Agent protocol) lets agents call other agents as if they were tools
  • Pattern: orchestrator agent delegates to specialist agents
from strands import Agent
from strands_tools import agent_tool

# Specialist: HR agent
hr_agent = Agent(
    system_prompt="You are an HR specialist. Answer only HR-related queries.",
)

# Make the HR agent callable as a tool
@agent_tool
def call_hr_agent(query: str) -> str:
    """Route HR-related questions to the HR specialist agent."""
    return hr_agent(query)

# Orchestrator
orchestrator = Agent(
    system_prompt="You are a company assistant. Route questions to the right specialist.",
    tools=[call_hr_agent]
)

orchestrator("What's the vacation policy?")
  • Each sub-agent has its own system prompt, model, and tool set — separation of concerns
  • The orchestrator's LLM decides when to delegate based on the tool docstrings
  • Agents can call other agents recursively — but watch for infinite loops

Lab 6 — Observability with Langfuse + Ragas

  • Strands emits OpenTelemetry (OTel) traces automatically — plug in any OTel-compatible backend
  • Langfuse: open-source LLM observability platform — traces, spans, cost tracking, latency
  • Ragas: evaluation framework for RAG and agent pipelines — measures faithfulness, relevance, etc.
import os
from strands import Agent
from langfuse.callback import CallbackHandler

# Set LANGFUSE_PUBLIC_KEY, LANGFUSE_SECRET_KEY, LANGFUSE_HOST env vars
langfuse_handler = CallbackHandler()

agent = Agent(
    system_prompt="You are a research assistant.",
    callback_handler=langfuse_handler
)

response = agent("Summarize the key themes of transformer architecture.")
# All tool calls, LLM calls, latency, tokens auto-tracked in Langfuse
  • Ragas evaluation metrics for agents: task completion, tool call correctness, answer relevance
  • In production, run Ragas evals on sampled traces to catch quality regressions
Key insight: Observability is not optional for production agents. Without traces you cannot debug why an agent made a bad decision or called the wrong tool.

Key Takeaways

  • Labs 1–3 cover the foundation: get an agent running, switch models, access AWS
  • Lab 4 (MCP) is the key skill for Phase 1 — build MCP servers, connect them to agents
  • Lab 5 (A2A) previews Phase 2 — multi-agent delegation is the core architectural pattern
  • Lab 6 (observability) is non-negotiable for production — instrument from day one
  • The Strands progression is: basic agent → tools → AWS → MCP → multi-agent → observability
