Build Your First AI Agent with Strands

AWS Developers channel · Mar 2026 · YouTube ↗

Why LLMs Alone Fail

  • No tool use — LLMs generate text; they can't call APIs, run calculations, or open files
  • No workflow — Complex tasks need plan → execute → check → retry, not just one-shot generation
  • No reliability — Without memory/tracking, the model hallucinates when it can't find an answer
  • Analogy: a chatbot is a consultant (gives advice), an agent is a worker (gets things done)

What Is Agentic AI?

  • Goal-driven software that reasons → chooses actions → uses tools → adapts based on results
  • Still uses an LLM as the "brain" — tools are the "hands"
  • Key property: if a tool call fails, the agent adapts and finds another route (not just failing out)

Strands Agents SDK Overview

  • Open-source SDK from AWS — Python and TypeScript
  • Default model provider: Amazon Bedrock → claude-sonnet (requires AWS credentials)
  • Supports any model provider: OpenAI, Anthropic direct, Llama, custom endpoints
  • Built-in tool library: calculator, file ops, shell, web/HTTP, AWS services, Slack, browser, multimodal
  • Supports MCP servers (external tool discovery) and A2A (agent-to-agent communication)
Key insight: Strands abstracts the orchestration loop — you define model + tools + prompt, and Strands handles the reason/act/observe cycle.

How the Agentic Loop Works

  • User sends a prompt → agent passes it to the LLM
  • LLM reasons: "what tool do I need?" → selects a tool from its available set
  • Tool executes → result returned to agent → LLM reasons again
  • Loop continues until the agent can produce a final answer

Installation

pip install strands-agents strands-agents-tools
  • If using Bedrock (default): configure AWS credentials first (aws configure or env vars)
  • strands-agents-tools contains the built-in tool library (calculator, shell, file, web, etc.)

Quick Start Pattern

from strands import Agent
from strands.models import BedrockModel

model = BedrockModel(model_id="us.anthropic.claude-sonnet-4-5-20251001-v1:0")

agent = Agent(
    model=model,
    system_prompt="You are a helpful assistant."
)

agent("What is the weather today?")
  • Without tools, the agent is just a fancy LLM wrapper
  • The real power appears when you attach tools

Adding Tools

from strands import Agent, tool
from strands_tools import calculator, current_time

# Built-in tools
agent = Agent(
    system_prompt="You are a helpful assistant. Use tools for math and time.",
    tools=[calculator, current_time]
)

# Custom tool using @tool decorator
@tool
def get_weather(location: str) -> str:
    """Get current weather for a location. Returns sunny/cloudy/rainy."""
    return "sunny"  # replace with real API call

agent_with_custom = Agent(
    system_prompt="You are a weather assistant.",
    tools=[get_weather]
)
Important: The docstring on a custom tool is passed to the LLM so it knows when to call the tool. Write it clearly — describe what the tool does and what it returns.

Built-in Tool Categories

  • Computation: calculator, current_time
  • File system: file read/write/list
  • Shell: run commands in the agent's environment
  • Web: HTTP requests, DuckDuckGo search, browser control
  • AWS: use_aws — wraps Bedrock, S3, DynamoDB, Lambda, and more
  • Code: Python code interpreter
  • Agents: MCP client, A2A communication, agent orchestration
  • Multimodal: image/document understanding

Direct Tool Invocation (for debugging)

# Call a tool directly without going through the agent loop
agent.tool.calculator(expression="sin(45) + cos(45)")
  • Useful when you want to verify a tool works before wiring it to an agent

Model Providers

  • Default: BedrockModel with Claude Sonnet — requires AWS credentials
  • Swap the model parameter to use OpenAI, Anthropic API, Llama, Ollama, etc.
  • See Strands model providers docs ↗

Logging

import logging
logging.getLogger("strands").setLevel(logging.DEBUG)
logging.basicConfig(format="%(levelname)s | %(name)s | %(message)s")
  • DEBUG level shows tool discovery, model calls, and tool results
  • INFO is enough for most development; WARNING for production

Key Takeaways

  • Strands in 3 lines: import Agent → define tools → call agent with a prompt
  • LLM = reasoning engine; tools = capabilities; Strands = the loop that connects them
  • Start with built-in tools, then add custom tools as you hit their limits
  • Docstrings on tools matter — the LLM reads them to decide which tool to call
